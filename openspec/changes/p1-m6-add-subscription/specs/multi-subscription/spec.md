## MODIFIED Requirements

### Requirement: 订阅增删切换更新
订阅 tab 必须（MUST）提供：
- **添加（多来源）**：支持 ① URL（HTTP 拉取）② 直接粘贴 clash 配置内容（本地校验、不走 HTTP）③ 从剪贴板填入 ④ 扫码二维码填入；可设名称（默认 host）、自定义 User-Agent（默认 `clash.meta`）、自动更新间隔；经 clash 配置校验后入库并落盘配置正文；
- **删除**：同步删除其落盘配置；
- **切换当前**；
- **更新**：用该订阅的 User-Agent 重新拉取配置与流量信息。

删除当前选中订阅时，当前选中必须（MUST）转移到列表首项，或在列表空时清空。

#### Scenario: URL 添加经 fetch 校验入库
- **当** 输入合法订阅 URL 并添加
- **则** 经 HTTP 拉取（带所设 User-Agent）+ clash 配置校验通过后入库并落盘正文

#### Scenario: 直接内容添加本地校验
- **当** 粘贴原始 clash 配置内容（非 URL）并添加
- **则** 本地校验为合法 clash 配置后入库落盘（不发起 HTTP）；非法则报错不入库

#### Scenario: 从剪贴板/扫码填入
- **当** 点「从剪贴板」或「扫码」
- **则** 把剪贴板文本 / 扫得文本填入输入框，供用户确认后再添加

#### Scenario: 删除当前订阅转移选中
- **当** 删除当前选中订阅且列表仍有其他订阅
- **则** 当前选中转移到列表首项，且其落盘配置被删除

## ADDED Requirements

### Requirement: 订阅自动更新
每订阅可（MAY）设置更新间隔（关闭，或 ≥5 分钟）。应用启动时必须（MUST）对**到期**订阅（间隔 > 0、有 URL、且距上次更新时间 ≥ 间隔）自动重新拉取并刷新落盘正文与流量信息、记录本次更新时间；纯内容订阅（无 URL）不得（MUST NOT）参与自动更新。自动更新失败不得（MUST NOT）阻塞应用启动或清空既有配置。

#### Scenario: 到期订阅启动时自动更新
- **当** 某订阅间隔为 6 小时、距上次更新已超 6 小时，应用启动
- **则** 该订阅被自动重新拉取，更新时间被刷新为当前

#### Scenario: 未到期不更新
- **当** 某订阅距上次更新未达其间隔
- **则** 启动时不自动更新该订阅

#### Scenario: 纯内容订阅不自动更新
- **当** 订阅为直接粘贴内容（无 URL）
- **则** 不论间隔设置，启动时均不对其发起自动更新
