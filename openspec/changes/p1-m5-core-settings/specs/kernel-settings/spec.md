## MODIFIED Requirements

### Requirement: 运行参数热改
内核设置页必须（MUST）承载运行参数偏好（日志级别 / IPv6 / 统一延迟 / TCP 并发 / 域名嗅探 / 延迟测试 URL·超时 / 局域网代理共享），其控件**未连接时也可设**（不再灰置「连接后可调」）；这些参数为预设偏好，改后应当（SHALL）提示「需重连生效」（不再依赖连接后经 external-controller 热改）。运行模式不在内核设置（在连接首页，连接中仍可热切）。偏好的持久化与连接时注入由 `run-params` 能力定义。维护动作（更新 GEO / 清缓存）作用于运行中内核，仍仅连接后可用。

#### Scenario: 未连接也可设运行参数
- **当** 未连接状态下进入内核设置
- **则** 日志级别 / IPv6 / 统一延迟 / TCP 并发 / 域名嗅探等控件可用（非灰置），修改即存为偏好

#### Scenario: 改参数提示重连生效
- **当** 在内核设置修改任一运行参数
- **则** 提示「需重连生效」，下次连接经配置注入生效

#### Scenario: 维护动作仍连接后可用
- **当** 未连接状态下查看维护动作（更新 GEO / 清缓存）
- **则** 这些动作灰置（连接后才可用），因其作用于运行中内核

#### Scenario: 运行模式不在内核设置
- **当** 查看内核设置的运行参数
- **则** 不含运行模式（运行模式在连接首页调节）
