## ADDED Requirements

### Requirement: 设置导航与应用设置
应用应当（SHALL）提供独立「设置」入口（底部导航第 4 个 tab），含主题模式切换（亮 / 暗 / 跟随系统）并持久化，以及关于信息（app 版本、mihomo 内核版本、GPL-3.0 许可、日志查看入口）。

#### Scenario: 切换主题并持久化
- **当** 用户在设置页切换主题模式
- **则** 界面即时应用新主题，且重启 app 后仍保持该选择

#### Scenario: 展示关于信息
- **当** 用户进入「关于」
- **则** 显示 app 版本、mihomo 内核版本与 GPL-3.0 许可，并提供日志查看入口

### Requirement: 订阅配置原文查看
应用应当（SHALL）提供对已获取订阅配置**原文**的只读查看，支持搜索、复制与导出；不得（MUST NOT）在此编辑或覆写配置（以订阅源为准）。展示的是订阅原文，不含内核运行时注入的覆写项。

#### Scenario: 查看订阅配置原文
- **当** 已获取订阅配置且用户打开配置查看
- **则** 只读展示完整 YAML 正文，可搜索关键词、选中复制、导出分享

#### Scenario: 尚未获取配置
- **当** 尚未获取任何订阅配置
- **则** 显示空态，提示先「获取配置」

### Requirement: 分流规则查看
应用应当（SHALL）在内核运行时经 external-controller 展示当前生效的分流规则列表，支持搜索；内核未运行或无数据时应当（SHALL）显示空态而非报错。

#### Scenario: 展示生效规则
- **当** 内核运行且用户打开规则查看
- **则** 列表展示生效规则（类型 / 匹配内容 / 出站），并可按关键词搜索

#### Scenario: 未连接空态
- **当** 内核未运行
- **则** 显示空态，提示先连接
