## ADDED Requirements

### Requirement: 多订阅模型与持久化
应用必须（MUST）支持多条订阅：每条含 id / 名称 / url / 流量·到期信息，配置正文按 id 落盘；订阅列表与当前选中（currentId）必须（MUST）持久化，重启保持。

#### Scenario: 添加订阅入库并持久化
- **当** 添加一条订阅（名称 + url）并重启应用
- **则** 读回的订阅列表含该条、其 url 与名称一致

#### Scenario: 当前选中持久化
- **当** 设置某订阅为当前并重启
- **则** 当前选中仍为该订阅

### Requirement: 订阅增删切换更新
订阅 tab 必须（MUST）提供：添加（url + 名称，经 fetch 与 clash 配置校验后入库）、删除（同步删除其落盘配置）、切换当前、更新（重新拉取配置与流量信息）。删除当前选中订阅时，当前选中必须（MUST）转移到列表首项，或在列表空时清空。

#### Scenario: 添加经校验入库
- **当** 输入合法订阅 url 并获取
- **则** 校验通过后入库并落盘配置正文

#### Scenario: 切换当前订阅
- **当** 点击某非当前订阅
- **则** 其成为当前选中

#### Scenario: 删除当前订阅转移选中
- **当** 删除当前选中订阅且列表仍有其他订阅
- **则** 当前选中转移到列表首项，且其落盘配置被删除

#### Scenario: 更新订阅重拉
- **当** 对某订阅点更新
- **则** 重新拉取其配置正文与流量·到期信息

### Requirement: 连接使用当前订阅
连接必须（MUST）使用当前选中订阅的配置正文；无当前选中订阅时，连接入口必须（MUST）禁用并提示先添加并选择订阅。

#### Scenario: 有当前订阅可连接
- **当** 存在当前选中订阅
- **则** 连接使用其配置正文启动内核

#### Scenario: 无当前订阅禁用
- **当** 无任何订阅或未选中当前
- **则** 连接入口禁用并提示去订阅页添加

### Requirement: 切换订阅提示重连
切换当前订阅时若隧道已连接，应当（SHALL）提示需重连生效，不得（MUST NOT）自动断开当前连接。

#### Scenario: 已连接切换提示重连
- **当** 已连接状态下切换当前订阅
- **则** 提示「重连生效」，当前连接不被自动断开
