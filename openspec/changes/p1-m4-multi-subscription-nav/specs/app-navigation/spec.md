## MODIFIED Requirements

### Requirement: 三层底部导航
应用底部导航必须（MUST）为三层：连接 / 订阅 / 设置。切换底部 tab 应当（SHALL）保留各 tab 状态，不中断已建立的数据流（如监控流量/日志订阅）。

#### Scenario: 三 tab 呈现
- **当** 应用启动进入主框架
- **则** 底部导航显示连接、订阅、设置三个 tab，无独立的节点/监控/内核设置底部 tab

#### Scenario: 切换底部 tab 保留状态
- **当** 在已连接状态从连接 tab 切到订阅再切回
- **则** 监控数据流/节点列表等状态保留、不重新初始化

### Requirement: 设置与内核设置分工
应用层项（外观主题 / 按需连接 / 关于：app 版本·开源许可）必须（MUST）位于设置 tab；内核设置必须（MUST）作为设置 tab 的二级入口（push 进入），不再是底部 tab；内核相关项（日志级别/IPv6 等运行参数、维护动作、配置查看、分流规则、内核版本、日志）位于内核设置二级页。

#### Scenario: 设置 tab 含应用层项与内核设置入口
- **当** 进入设置 tab
- **则** 含外观主题、按需连接、关于，以及「内核设置」入口

#### Scenario: 内核设置为二级页
- **当** 在设置 tab 点击「内核设置」
- **则** push 进入内核设置页（含日志级别/IPv6/维护/配置规则/内核版本/日志）
