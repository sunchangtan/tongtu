## ADDED Requirements

### Requirement: 三层底部导航
应用底部导航必须（MUST）精简为三层：连接 / 设置 / 内核设置。切换底部 tab 应当（SHALL）保留各 tab 状态，不中断已建立的数据流（如监控流量/日志订阅）。

#### Scenario: 三 tab 呈现
- **当** 应用启动进入主框架
- **则** 底部导航显示连接、设置、内核设置三个 tab，无独立的「节点」「监控」底部 tab

#### Scenario: 切换底部 tab 保留状态
- **当** 在已连接状态从连接 tab 切到设置再切回
- **则** 监控数据流/节点列表等状态保留、不重新初始化

### Requirement: 连接首页整合节点与监控
连接 tab 必须（MUST）用顶部 TabBar 承载「连接 / 节点 / 监控」三子页（复用现有页面逻辑），节点与监控不再是独立底部 tab。切换子页应当（SHALL）保留各子页状态。

#### Scenario: 连接 tab 含三子页
- **当** 进入连接 tab
- **则** 顶部 TabBar 含连接、节点、监控三个可切换子页

#### Scenario: 切换子页保留状态
- **当** 在监控子页订阅数据流后切到节点子页再切回
- **则** 监控数据流不中断、不重新订阅

### Requirement: 设置与内核设置分工
应用层项（外观主题 / 按需连接 / 关于：app 版本·开源许可）必须（MUST）位于设置 tab；内核相关项（运行参数 / 维护动作 / 配置查看 / 分流规则 / 内核版本 / 日志）必须（MUST）位于内核设置 tab。

#### Scenario: 设置 tab 含应用层项
- **当** 进入设置 tab
- **则** 含外观主题、按需连接、关于（app 版本/许可），不含配置查看/分流规则/内核版本/日志

#### Scenario: 内核相关归内核设置
- **当** 进入内核设置 tab
- **则** 含运行参数、维护动作、配置查看、分流规则、内核版本、日志
